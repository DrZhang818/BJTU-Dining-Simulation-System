## 二、对成员A、成员C 已交付内容的简要评估

### 成员A（张恩崎）核心仿真模块

- **架构**：`bdss::core` / `bdss::utils` 命名空间分层清晰，`include/` 与 `src/` 对称，`Config / Student / Window / Canteen / SimulationEngine` 职责单一。
- **状态机完整**：`StudentState` 覆盖 Arrived → Queuing → Serving → WaitingForSeat → Dining → Left；时间字段用 `std::optional<int>`，未发生时返回 -1。
- **窗口效率模型**：`Window` 用 `workAccumulator_` 累加 `efficiency_`，效率 0.8/0.9/1.0/1.1/1.2 循环分配。
- **Engine 调度顺序自洽**：tick 内"生成 → 服务 → 就餐 → 寻座 → 记录"；`isFinished()` 在仿真窗口结束且系统清空后才返回 true，不漏统计。
- **CMake 配置专业**：核心代码独立成 `BDSS_CORE_SOURCES` 列表，`BDSS` 与 `BDSS_CoreTest` 共用，C++23 / `CXX_EXTENSIONS OFF`。
- **可优化点（已记录）**：
  - `Config::loadFromFile` 是手写的字符串查找解析器，不支持嵌套 / 注释，建议后续换 `QJsonDocument`。
  - `Canteen::trySeat` 是 O(rows×cols) 线性扫描，10×10 没问题，调到 50×50 再优化。
  - `seatWaitingStudents()` 一旦坐不下就 break，语义稍隐晦但逻辑无误。

**结论**：核心闭环完整，接口清晰，本轮 GUI 直接基于现有接口对接即可。

### 成员C（孜克茹拉·阿不都克依木）测试与统计模块

- **`RandomGenerator`**：`thread_local` 引擎 + 全静态接口，对 `lambda<=0` / `stddev<=0` 都做了守卫。
- **`StatisticsLogger`**：完成人数、最大排队长度、最大等座人数、平均等待 / 服务 / 就餐 / 总耗时、平均座位利用率全覆盖；CSV 表头干净。
- **`tests/test_core.cpp`**：用 `assert` 覆盖 Student / Window / Canteen / RandomGenerator / SimulationEngine 五个场景。
- **可优化点（已记录）**：
  - CSV 没记录窗口级数据（每个窗口的队列长度），GUI 想画窗口热力图要自己采样 → **本轮 GUI 直接调 `engine->getWindowQueueLengths()` 解决**。
  - 没有按学生粒度的明细 CSV，分布直方图要做的话需要扩展。
  - `getAverageSeatUtilization()` 是按 tick 平均，会被仿真初期的低利用率拉低。

**结论**：统计字段够用，CSV schema 兼容 QtCharts；`getRecords()` 已扩展暴露 TickRecord 序列，下一轮做曲线图直接用。

### 3.4 编译过程中遇到的问题

1. **首次 `cmake ..`** 缺少 Qt 路径 → 加 `-DCMAKE_PREFIX_PATH=/opt/homebrew/lib/cmake` 解决。
2. **`MainWindow::setRunningUi`** 用 `for (QWidget* w : { spinXxx, ... })` 时 braced-init-list 无法推导出统一类型（`QSpinBox*` 与 `QDoubleSpinBox*` 不兼容）→ 改成显式 `const QWidget* const arr[]` 数组绕过。
3. **clangd 一次性报 87 条 lint 错误**（QWidget / Config.h 找不到等）→ 是 IDE 没读到 CMake 的 include 路径，跑 `cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON .` 并把 `compile_commands.json` 软链到根目录后即解决；**实际 `make` 全程 0 错 0 警告**。

## 五、运行说明

### 启动 GUI

**必须**在项目根目录（不是 `build/` 内）启动，让程序能找到 `resources/default_config.json`：

```bash
cd /Users/wangsheng/BJTU-Dining-Simulation-System-main
./build/BDSS
```

### 操作流程

1. 启动后默认进入【② 实时监控】Tab，画布提示"尚未启动仿真"。
2. 切到【① 参数配置】Tab，按需调整窗口数 / 餐桌行列 / 到达率 / 高峰倍率等参数。
3. 点工具栏的【▶ 开始】，自动跳回实时监控页，QTimer 以 50ms / 帧驱动 `engine.tick()`。
4. 拉动【仿真速度】滑块切换 1× / 5× / 20× / 100× / 500×（即每帧 tick 多少次）。
5. 任何时候可点【⏸ 暂停】、【⏭ 单步】（暂停时调试用）、【↺ 重置】（清空引擎状态）。
6. 仿真结束时自动停止 QTimer，状态栏显示完成学生数。

### 头模式（保留供成员A、成员C 调试）

```bash
./build/BDSS --headless
# 或 -c / --no-gui
```

行为与重写前完全一致：跑完默认 1 小时仿真 → 控制台打印 summary → 写出 `simulation_log.csv`。

## 六、对外接口契约（与成员A、成员C 协作）

GUI 仅通过以下只读接口访问核心层，**不修改 engine 内部状态**：

| 接口 | 用途 | 调用频率 |
|---|---|---|
| `engine->tick()` | 推进 1 秒仿真 | 每帧 N 次（N = ticksPerFrame） |
| `engine->isFinished()` | 判断终止 | 每次 tick 前后 |
| `engine->getCurrentTime()` | 数字面板"仿真时间" | 每帧 |
| `engine->getTotalSimulationTime()` | 数字面板分母 | 每帧 |
| `engine->getTotalQueueLength()` | 数字面板"窗口排队总数" | 每帧 |
| `engine->getWaitingForSeatCount()` | 数字面板"等座人数" | 每帧 |
| `engine->getOccupiedSeats()` / `getTotalSeats()` | 数字面板"已就座" | 每帧 |
| `engine->getSeatUtilization()` | 数字面板"座位利用率" | 每帧 |
| `engine->getWindowQueueLengths()` | 左半区每个窗口的队列条长度 | 每帧 |
| `engine->getWindowEfficiencies()` | 左半区窗口标签 ×0.8 / ×1.0 等 | 引擎构建后一次 |
| `engine->getCanteen().getSeatSnapshot()` | 右半区 rows×cols 占用矩阵（学生 ID，0 = 空） | 每帧 |
| `engine->getStatistics().getFinishedCount()` | 数字面板"已离开学生" | 每帧 |
| `engine->getStatistics().getRecords()` | **下一轮**统计图表 Tab 用 | 仿真结束 |

新增的访问器全部是 const 引用，零拷贝，符合"GUI 只读"原则；如果成员A 要重构 `Canteen` 内部存储，只要保持 `getSeatSnapshot()` 返回 `vector<vector<int>>` 不变即可，GUI 不受影响。
