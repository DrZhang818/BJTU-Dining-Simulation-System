# BDSS 项目优化说明

## 1. 构建系统优化

- 新增 `BDSSCore` 静态库，避免主程序和测试程序重复维护源文件列表。
- 保留 Qt GUI 自动检测逻辑；无 Qt 环境时输出提示并继续构建 headless 版本。
- 增加 `enable_testing()` 和 `add_test()`，支持 `ctest` 标准测试流程。
- 增加严格编译告警选项，提前暴露潜在问题。

## 2. 核心模型优化

- 将学生生命周期明确为 `Arrived -> Queuing -> Serving -> WaitingForSeat -> Dining -> Left`。
- 窗口模型区分可见排队人数和实际工作负载，分配新学生时优先选择工作负载最小窗口。
- 餐桌模型使用二维座位矩阵，实时维护已占用座位数，避免每次统计都全表扫描。
- 高峰期到达模型使用泊松分布，`arrivalRate` 按“每分钟学生数”解释，更符合食堂到达过程。

## 3. 统计能力优化

- `StatisticsLogger` 同时记录时间序列快照和最终汇总指标。
- CSV 字段固定、简洁，便于 Excel/WPS 直接画图。
- 汇总指标包括完成学生数、最大排队人数、最大等座人数、平均排队等待、平均等座等待、平均服务时间、平均总停留时间、平均座位利用率。

## 4. 参数与可复现性优化

- 配置文件读取后统一校验，避免非法输入进入仿真内核。
- 支持命令行覆盖随机种子和仿真时长。
- 默认随机种子为 42，便于课堂验收和报告复现。

## 5. 测试优化

当前测试覆盖：

1. 学生状态切换与时间计算。
2. 窗口排队和服务完成逻辑。
3. 餐桌分配、就餐推进和座位释放。
4. 配置解析和非法配置校验。
5. 随机数范围和泊松分布生成。
6. 完整仿真流程终止条件。
7. 统计日志 CSV 导出。
8. 画像类型序列化与解析。
9. 学生画像与偏好窗口字段。
10. 窗口品类。
11. 偏好关闭时行为回归一致。
12. 偏好开启冒烟测试。
13. 品类数量不匹配校验。

## 6. 就餐偏好系统（v2.1 新增）

- **就餐者画像**：本科生/研究生/教职工，可配置比例与就餐时长倍率。
- **窗口品类偏好**：窗口分配品类标签，学生按"队列长度 × 权重 − 命中偏好奖励"的评分选择窗口。
- **数据流**：`Config` 承载参数 → `SimulationEngine` 消费 → `StatisticsLogger` 记录。
- **向后兼容**：`enablePreferences=false` 时引擎行为与 v2.0 逐行等价，`seed=42` 结果不变。

## 7. GUI 统计图表（v2.1 新增）

- 统计图表 Tab 使用 `QChartView` + `QLineSeries` 绘制三条实时曲线：排队总人数、等座人数、座位利用率。
- 双 Y 轴设计：左侧为人数、右侧为利用率（0–100%）。
- 仿真结束时自动更新并切换至图表 Tab，下方展示完整汇总面板（完成数、最大队长、平均等待等 6 项指标）。
- 工具栏「导出 CSV」按钮仿真结束后可用，通过 `StatisticsLogger::exportCSV()` 导出。

## 8. 后续可扩展方向

- 将时间步长模型升级为真正的事件优先队列模型，进一步提升大规模仿真性能。
- 增加排队耐心与放弃/换队行为模型。
- 增加打包/堂食模式，结伴就餐与相邻座位。
- 增加座位翻台/清洁机制。
- 增加多峰到达曲线、菜品售罄限量。
- 增加参数灵敏度分析，自动输出窗口数量和座位数量的优化建议。
