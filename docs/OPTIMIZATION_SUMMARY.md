# BDSS 项目优化说明

## 1. 构建系统优化

- 新增 `BDSSCore` 静态库，避免主程序和测试程序重复维护源文件列表。
- 保留 Qt GUI 自动检测逻辑；无 Qt 环境时输出提示并继续构建 headless 版本。
- 增加 `enable_testing()` 和 `add_test()`，支持 `ctest` 标准测试流程。
- 增加严格编译告警选项，提前暴露潜在问题。

## 2. 核心模型优化

- 将学生生命周期明确为 `Arrived -> Queuing -> Serving -> WaitingForSeat -> Dining -> Left`。
- 窗口模型区分可见排队人数和实际工作负载，分配新学生时优先选择工作负载最小窗口。
- 餐桌模型使用二维座位矩阵，实时维护已占用座位数，避免每次统计都全表扫描。
- 高峰期到达模型使用泊松分布，`arrivalRate` 按“每分钟学生数”解释，更符合食堂到达过程。

## 3. 统计能力优化

- `StatisticsLogger` 同时记录时间序列快照和最终汇总指标。
- CSV 字段固定、简洁，便于 Excel/WPS 直接画图。
- 汇总指标包括完成学生数、最大排队人数、最大等座人数、平均排队等待、平均等座等待、平均服务时间、平均总停留时间、平均座位利用率。

## 4. 参数与可复现性优化

- 配置文件读取后统一校验，避免非法输入进入仿真内核。
- 支持命令行覆盖随机种子和仿真时长。
- 默认随机种子为 42，便于课堂验收和报告复现。

## 5. 测试优化

当前测试覆盖：

1. 学生状态切换与时间计算。
2. 窗口排队和服务完成逻辑。
3. 餐桌分配、就餐推进和座位释放。
4. 配置解析和非法配置校验。
5. 随机数范围和泊松分布生成。
6. 完整仿真流程终止条件。
7. 统计日志 CSV 导出。

## 6. 后续可扩展方向

- 将时间步长模型升级为真正的事件优先队列模型，进一步提升大规模仿真性能。
- 增加多窗口类型、菜品偏好、同伴结伴就餐等行为模型。
- 在 Qt GUI 中补全统计图表页，直接展示 CSV 趋势曲线。
- 增加参数灵敏度分析，自动输出窗口数量和座位数量的优化建议。
